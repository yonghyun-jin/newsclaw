# NewsLaw Documentation

## 📁 Folder Structure

### `/docs/features/` 
**Recommended for feature prompts and specifications**
- `feature-1-auth.md` - Authentication feature prompt
- `feature-2-dashboard.md` - Dashboard feature prompt  
- `feature-3-api.md` - API endpoints feature prompt

### `/docs/prompts/`
**For AI/development prompts and templates**
- `coding-prompt-templates.md` - Reusable prompt templates
- `feature-development-workflow.md` - Step-by-step processes

### `/planning/`
**For high-level planning and roadmaps**
- `roadmap.md` - Product roadmap
- `architecture.md` - Technical architecture decisions

## 🎯 Recommended Structure for Feature Prompts

```
docs/features/
├── 01-authentication.md
├── 02-user-dashboard.md  
├── 03-file-upload.md
├── 04-notifications.md
└── template.md
```

Each feature file should include:
- **Objective**: What this feature accomplishes
- **Requirements**: Detailed specifications
- **Technical Stack**: Technologies needed
- **API Endpoints**: Backend requirements
- **UI/UX**: Frontend components needed
- **Testing**: Acceptance criteria
